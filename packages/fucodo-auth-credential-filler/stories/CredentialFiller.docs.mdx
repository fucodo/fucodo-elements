## Description

A flexible Web Component that loads credentials from a URL and/or uses inline JSON defaults. It provides a searchable dropdown to fill username and password fields, supporting grouped entries and optional descriptions.

Tag name: `fucodo-auth-credential-filler`

## Features

- **URL & Inline Data**: Fetch credentials from a remote JSON endpoint and/or provide local fallbacks.
- **Grouped Entries**: Organise credentials into categories (e.g., Admins, Editors).
- **Searchable Dropdown**: Filter through credentials by typing in the username field.
- **Descriptions**: Optional secondary information for each entry.
- **Keyboard Support**: Navigate the dropdown using arrow keys and select with Enter.
- **Standard Web Component**: Works with any framework or pure HTML/JS.

## API Reference

### Attributes

| Attribute | Required | Default | Description |
|-----------|----------|---------|-------------|
| `url` | No | - | URL of a JSON endpoint. |
| `username-selector` | **Yes** | - | CSS selector for the username input field. |
| `password-selector` | **Yes** | - | CSS selector for the password input field. |
| `group-key` | No | - | Property name for group headers in the dropdown. |
| `description-key` | No | - | Property name for a secondary label below the username. |
| `username-key` | No | `username` | Property name to be used as the username. |
| `password-key` | No | `password` | Property name to be used as the password. |
| `label-key` | No | `username` | Property name to be displayed as the primary label in the dropdown. |

### Events

| Event | Detail | Description |
|-------|--------|-------------|
| `credential-filler:ready` | `{ count, source }` | Fired when data is loaded. `source` is `'default'` or `'url'`. |
| `credential-filler:select` | `{ username, entry, source }` | Fired when an entry is selected. |
| `credential-filler:error` | `{ error, usingDefault }` | Fired when the URL fetch fails. |

## Theming

The dropdown is injected into the document body and can be styled via global CSS. It uses the following classes:

- `.cf-dropdown`: The main container.
- `.cf-group-header`: Headers for each group.
- `.cf-item`: Individual entries.
- `.cf-item-desc`: Secondary description text.
- `.cf-active`: Highlighted state (hover or keyboard focus).
- `mark`: Highlighted text matching the search query.
