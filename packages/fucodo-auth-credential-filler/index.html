<credential-filler>

Web Component — lädt Zugangsdaten von einer URL, unterstützt Gruppen und befüllt Username/Passwort-Felder per eigenem Dropdown.

Demo A — Default wird von URL ersetzt (mit Gruppen)

Initialisiere…
Sofort bereit mit Offline-Gruppe aus den Defaults. Nach dem Fetch erscheinen Admins / Editors / Viewers aus der Remote-URL.

Initialisiere…
Fetch schlägt fehl — System / Users aus den eingebetteten Defaults bleiben nutzbar.

Attribute

AttributPflichtStandardBeschreibung
urlnein—URL eines JSON-Endpoints. Ohne url werden nur Defaults genutzt.
username-selectorja—CSS-Selector des Benutzername-Inputs.
password-selectorja—CSS-Selector des Passwort-Inputs.
group-keynein—JSON-Property für die Gruppenzuordnung (z.B. "group" oder "role"). Ohne dieses Attribut erscheinen alle Einträge ohne Gruppenheader.
description-keynein—JSON-Property für eine optionale Beschreibung, die klein unter dem Benutzernamen angezeigt wird.
username-keyneinusernameJSON-Property-Name für den Benutzernamen.
password-keyneinpasswordJSON-Property-Name für das Passwort.
label-keyneinusernameJSON-Property, die im Dropdown als Label angezeigt wird.

Verwendung

<credential-filler
  url="https://api.example.com/credentials.json"
  group-key="group"
  description-key="description"
  username-selector="#user"
  password-selector="#pass"
></credential-filler>
// JSON-Format — description ist optional pro Eintrag:
[
  { "username": "alice", "password": "s3cr3t", "group": "Admins", "description": "Superadmin" },
  { "username": "bob",   "password": "hunter2", "group": "Admins"  },
  { "username": "carol", "password": "abc123",  "group": "Viewers", "description": "Testkonto" }
]
Einträge ohne group-Property erscheinen ohne Gruppenheader am Ende der Liste. Die Gruppenreihenfolge im Dropdown folgt dem ersten Auftreten im Array. Tippen filtert alle Gruppen gleichzeitig; leere Gruppen werden dabei ausgeblendet.

Events

EventdetailBeschreibung
credential-filler:ready{ count, source }Dropdown aufgebaut. source: 'default' oder 'url'. Feuert bis zu zweimal wenn Defaults + URL vorhanden.
credential-filler:select{ username, entry, source }Eintrag ausgewählt (Klick oder Enter).
credential-filler:error{ error, usingDefault }URL-Fetch fehlgeschlagen. usingDefault: true wenn Defaults weiterhin aktiv sind.
